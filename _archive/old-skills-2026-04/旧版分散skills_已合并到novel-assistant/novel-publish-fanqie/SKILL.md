# 小说发布到番茄技能

## 触发条件

- 用户说"发布到番茄"
- 用户说"番茄发布"
- 用户说"上传番茄"
- 用户说"发布章节"

## 工作流程

### 1. 定位项目根目录

```
搜索顺序：
1. 当前工作目录是否有 `项目文件/` 或 `project-files/`
2. 当前工作目录是否有 `正文卷/` 或 `chapters/`
3. 向上查找最近的项目文件集合
```

### 2. 读取小说信息

从项目文件中获取：
- 小说名称（从 `99_当前状态卡.md` 或文件夹名）
- 已完成的章节列表
- 章节文件路径

### 3. 导出章节为txt格式

将md格式的章节转换为番茄要求的txt格式：

**格式要求：**
- 文件名：`001 第1章.txt`、`002 第2章.txt`（序号+空格+章节名）
- 内容：纯文本，移除markdown格式符号
- 编码：UTF-8

**转换规则：**
- 移除 `【第X章 XXX】` 格式的标题行（文件名已包含）
- 移除 `**粗体**` 的星号
- 移除 `*斜体*` 的星号
- 移除 `> 引用` 的箭头
- 保留段落换行
- 移除markdown链接，只保留文字

### 4. 放入番茄发布目录

目标目录结构：
```
fanqie_auto_publish/
├── chapters/
│   └── {小说名称}/
│       ├── 001 第1章.txt
│       ├── 002 第2章.txt
│       └── ...
├── uploaded/
│   └── {小说名称}/
└── state.json
```

### 5. 输出发布报告

```markdown
## 📤 番茄发布准备完成

### 小说信息
- **小说名称**：{名称}
- **导出章节**：X章
- **目标目录**：fanqie_auto_publish/chapters/{名称}/

### 已导出章节
- [x] 001 第1章.txt
- [x] 002 第2章.txt
- ...

### 下一步操作
1. 打开终端，进入 fanqie_auto_publish 目录
2. 运行 `python login.py` 进行扫码登录（首次使用）
3. 运行 `python publish.py` 开始自动发布
4. 选择小说序号，设置发布数量
5. 等待自动发布完成

### 命令速查
```bash
cd d:\AI\AI小说创作系统\fanqie_auto_publish
python login.py    # 首次登录
python publish.py  # 发布章节
```
```

## 可选配置

### 发布配置

在项目根目录创建 `番茄发布配置.md` 可自定义发布行为：

```markdown
# 番茄发布配置

## 是否自动调用发布脚本
自动发布：true / false（默认false，需手动确认）

## 每次发布章节数
发布数量：10（默认全部）

## 默认分卷
分卷：1（默认不分卷）
```

## 注意事项

### 首次使用

1. **安装依赖**：
```bash
pip install playwright pillow
playwright install chromium
```

2. **扫码登录**：
```bash
cd fanqie_auto_publish
python login.py
```
登录成功后会生成 `state.json` 文件，后续无需重复登录。

### 发布限制

- 番茄平台每日发布有上限
- 建议每次发布10-20章
- 发布后章节会自动移动到 `uploaded/` 目录

### 章节格式

- 文件名必须包含章节号
- 文件名格式：`序号 章节名.txt`
- 示例：`001 第1章.txt`、`101 第101章.txt`

## 完整自动化流程

```
写小说 → 降重 → 同步项目文件 → 发布到番茄 → 自动上传
   ↓        ↓           ↓            ↓
正文卷/   混沌口语流  progress.md   chapters/书名/
```

## 故障排除

### 登录失败
- 检查 `state.json` 是否存在
- 重新运行 `python login.py` 扫码登录

### 发布中断
- 脚本支持断点续传
- 重新运行 `python publish.py` 会自动识别未完成章节

### 弹窗拦截
- 脚本内置弹窗斩杀器
- 如遇新弹窗，可手动点击后继续
