# ⛔ 六条铁律（每次输出必须遵守，违反任意一条 = 任务失败）

## 铁律 1：对话必须用双引号独立成段

**错误示范（绝对禁止）：**
```
没事，陈大山笑了笑，就是不想在城里待了。
大山！身后传来声音。
秀梅，陈大山停下脚步。
```

**正确示范（必须照此写）：**
```
"没事。"
陈大山摸了摸鼻子。
"就是不想在城里待了。"

"大山！"
身后传来声音。

"秀梅。"
陈大山停下脚步。
```

规则：
- 每一句人说出来的话独立成段
- **中文弯引号 `""` 优先**，英文直引号 `""` 也可以（auto_fix 脚本会自动转换成中文弯引号，不要为了纠结引号形式反复重写）
- 动作描写另起一段，不能和对话混在一行

## 铁律 2：禁用 AI 模板词，每词全书限用 ≤2 次

| 禁用词 | 超限后必须替换为（不重复） |
|---|---|
| 笑了笑 | 没说话 / 嘴角动了动 / 深吸了口气 / 摸了摸鼻子 / 把目光移开 |
| 点了点头 | 嗯了一声 / 收回目光 / 把话咽回去 / 眉头微皱 / 没接话 |
| 脸红了红 | 低下了头 / 咬住了下唇 / 耳根子有点热 / 转过身去 |
| 心里头一暖 | 鼻子有点酸 / 嗓子眼堵了什么 / 低头没说话 / 吸了吸鼻子 |
| 心里头那个高兴 | 嘴角压不住 / 差点笑出声 / 手都有点抖 |
| 眼睛一亮 | 盯着看了好几遍 / 手里的动作停了 / 眉头挑了一下 |
| 眼睛都直了 | 愣在原地 / 嘴张了张没说出话 / 手一抖差点掉了 |
| 激动得心跳加速 | 手心全是汗 / 深吸了两口气 / 握了握拳头 |

**写作时每次想写这些词，先停一下，换成替换词。同一替换词一章内也只能用一次。**

## 铁律 3：每段必须有口语化词

至少包含以下之一：`呢 / 吧 / 嘛 / 咋 / 啥 / 呗 / 琢磨 / 寻思`

**⚠️ 反刷分规则（新增）**：禁止用单一口语词堆砌。同一个词（如"寻思着"/"琢磨着"）在全章口语词中占比不得超过 **40%**。

**错误示范（刷分，脚本会 FAIL）：**
```
心里头咯噔一下，寻思着是不是出事了。
握紧拳头，寻思着这次不能忍。
蹲下来，寻思着得想办法。
叹了口气，寻思着还是得去一趟。
```
（4 句话全用"寻思着"，这会被反刷分检测抓住）

**正确做法（混用多种口语词）：**
```
心里头咯噔一下，盘算着是不是出事了。
握紧拳头，合计着这次不能忍。
蹲下来，琢磨着得想办法。
叹了口气，心里头清楚还是得去一趟。
```

口语化是**自然表达**，不是"某一个词的频率"。口语词清单：`呢/吧/嘛/咋/啥/呗/啊/呀/咯/喽/哟/哼/琢磨/寻思/盘算/合计/玩意儿/咱们/那小子/这小子/那会儿/可不是/不是嘛`——每种都用一点，不要偏科。

## 铁律 4：每章主角必须遭遇一次挫折

禁止"做事 → 成功 → 完美收尾"。必须有以下至少一种：
- 行动挫折：努力了但没全成（"查了半天地契，只确认是假的，没拿到直接证据"）
- 人际挫折：好心被误解
- 损失挫折：付出代价（用灵珠累得发烧）
- 意外挫折：计划被打乱
- 能力挫折：金手指副作用发作

## 铁律 5：反派必须有实质行动，禁止只口头威胁

每次反派出场必须至少选一个实质行动：
- 毁药田 / 砸药铺招牌
- 买通镇上药铺拒收主角药材
- 向大队举报主角"搞封建迷信"
- 散布"主角的药吃死人"谣言
- 带人截断主角上山或进镇的路
- 拉拢主角身边的人（林秀梅家长、村长等）
- 威胁村民不准找主角看病

**反派至少要有 3 次成功阻击主角（不是失败）**。每次失败后必须升级手段。

## 铁律 6：金手指必须有代价

每次用百草灵珠都必须写出以下至少一项：
- 身体代价：疲惫、发烧、手抖、视力模糊
- 时间代价：需要恢复期，错过重要事件
- 隐患代价：被人察觉异常，引起怀疑
- 资源代价：灵珠能量消耗，紧急时刻用完
- 反噬代价：超限导致副作用

**代价必须在后续剧情中真的产生影响，不能下一章就完全恢复。**

# ⛔ 强制脚本守门流程（不跑不许交付）

写完每一章后，**必须**在 Bash 工具里按顺序执行两个脚本：

## 步骤 A：自动修复

```bash
python D:/AI/AI小说创作系统/novel-assistant/scripts/auto_fix_chapter.py --project D:/AI/AI小说创作系统/山村小神医 --chapter <章节号>
```

自动补齐引号、替换 AI 模板词。
**注意**：`--project` 必须用**绝对路径**（因为 Trae 的工作目录不固定）。
只想预览不改文件时加 `--dry-run` 参数。

## 步骤 B：健康检查

```bash
python D:/AI/AI小说创作系统/novel-assistant/scripts/chapter_health_check.py --project D:/AI/AI小说创作系统/山村小神医 --chapter <章节号>
```

**注意**：`--project` 必须用**绝对路径**。

**当前 CRITICAL 阈值**（违反任一项 → 退出码 1 → 必须修复）：
- 引号对数 ≥ 6（中文/英文引号都算，auto_fix 自动转换）
- AI 模板词 总数 ≤ 8 / 单词 ≤ 3 次
- 口语化频率 ≥ 1.0/百字
- 预估 AI 率 ≤ 70%

**WARNING 阈值**（不阻断交付，建议修复）：
- AI 率 55-70%
- 字数偏离 2200-3400 超过 20%
- 冲突密度：距最近冲突 ≥ 3 章

脚本已放宽阈值适配弱模型（DeepSeek/MiniMax/Qwen），不要再无限重写追求完美分数。

**退出码 0**：通过，可交付。
**退出码 1**：CRITICAL FAIL，**必须重写整章并重跑脚本**，直到返回 0。
**退出码 2**：WARNING，建议修复。

## 绝对禁止

- ❌ 跳过脚本、不跑就说"已完成"
- ❌ 脚本返回 1 但坚持说"通过了"
- ❌ 自己给自己打 9 分（真分数由脚本给）
- ❌ 告诉用户"脚本没必要跑"
- ❌ 认为"我写得很好脚本不会失败"——脚本才是真相

**你的每一次"章节完成"输出都必须附带脚本的真实退出码截图/文本。**

# 七步工作流（严格按顺序）

## 步骤 1：读上下文

**P0 必读：**
- `项目文件/99_当前状态卡.md`
- `项目文件/task_plan.md`
- 前一章正文最后 500 字 + 前两章正文最后 200 字

**P1 按需读：**
- `项目文件/pending_hooks.md` — 关注 OVERDUE/URGENT 标记
- `项目文件/character_states.md`
- `项目文件/plot_threads.md`
- `项目文件/chapter_analysis.md`

## 步骤 2：构思（必须在动笔前回答完）

明确列出：
- **冲突点**：本章什么冲突？
- **主角挫折**：主角在哪里失败/付代价？（必填）
- **反派实质行动**：反派做了什么破坏？（必填，参考铁律 5）
- **金手指代价**：用了灵珠吗？代价是什么？（如用）
- **章末钩子**：留什么悬念？

## 步骤 3：写初稿

- 展示不讲述：用动作+反应代替"他很愤怒"
- 每段 1-3 句，对话独立成段用 `""`
- 禁止连续 2 章用相同结构（如"走进院子→众人围过来→坐下→对话"）
- 代词交替：陈大山 ↔ 那小子 ↔ 大山；林秀梅 ↔ 秀梅 ↔ 林老师

## 步骤 4：四阶段降重（BypassAIGC）

1. **润色**：扩展动词（"说" → "开口说了一句" / "冷笑"）
2. **增强**：加 `呢/吧/嘛/咋/啥`，打破对称句式
3. **场景降重**：对话加动作，心理用具象化
4. **整体润色**：删过度工整句，加真实不完美

## 步骤 5：强制跑脚本（见上方"强制脚本守门流程"）

返回非零 → 修复 → 重跑 → 直到返回 0。

## 步骤 6：章节分析

追加到 `项目文件/chapter_analysis.md`：
1. hooks（开篇+章末钩子）
2. foreshadows（伏笔：新埋/推进/回收）
3. plot_points（主要事件 2-3 个）
4. conflicts（类型/强度/结果）
5. character_changes（角色状态变化）
6. quality_score（节奏/冲突/角色/爽感/降重，1-10 自评）

任何维度 ≤ 5 → 改后重跑脚本。

## 步骤 7：更新状态文件

- `progress.md` — 标记章节完成
- `99_当前状态卡.md` — 更新时间/场景/主角状态
- `pending_hooks.md` — 新伏笔 + OVERDUE/URGENT 标记
- `task_plan.md` — 任务完成
- `power_tracking.md` — 战力变化
- `timeline.md` — 时间线
- `resource_ledger.md` — 资源
- `character_states.md` — 角色状态快照 + 亲密度
- `plot_threads.md` — 线索推进
- 每 10 章一次：`arc_summary.md` 弧线摘要

# 弱模型防漂移提示（给你自己看）

你是 MiniMax/DeepSeek/Qwen 等弱模型，写到 2000 字后可能会忘记铁律。请遵守以下自救策略：

1. **每写完 500 字，回头扫一眼铁律 1（引号）**——是不是每句对话都用 `""` 或 `""` 独立成段？
2. **遇到"笑了笑"就停**——换成替换词，查一下本章是否已用过该替换词。
3. **写反派场景时，先想一下铁律 5 的六选一清单**，挑一个实质行动写出来。
4. **交付前最后一步永远是跑脚本**，不是自检打钩。
5. **如果脚本返回 1，先看脚本建议的修复路径**——通常跑 auto_fix 就能解决引号+模板词问题。
6. **用户问进度时，以脚本退出码为准**，不要用"我觉得""我认为"。
