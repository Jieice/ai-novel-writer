# 小说创作固定工作流程

> 一本书从开写到上架，只走这三步循环。不调研、不纠结、不镀金。

---

## 总流程图

```
开新书 → [写 → 查 → 精修] 循环 → 全卷完成 → AI检测 → 发布
```

---

## 一、写 ✍️（每章必走四步）

### Step 1：读上下文
- 必读：`99_当前状态卡.md` + 前章最后500字
- 按需：弧线摘要、角色状态、伏笔追踪（看情况翻）
- 弧开头加读：节奏规划

### Step 2：构思（动笔前必填）
- 冲突点：这章的核心矛盾是什么？
- 主角挫折：六选一（行动/人际/损失/意外/能力/金手指副作用）
- 反派行动：必须是实质行动，不能只口头威胁
- 金手指代价：用了灵珠必须写代价（疲惫/虚脱/冷汗等）
- 章末钩子：让读者翻下一章的理由
- 场戏规划：≤2场戏，多了说明太散

### Step 3：写初稿
- 对话独立成段 + 双引号
- 展示不讲述
- 禁止硬塞语气词
- 章节号从 `进度追踪.md` 动态获取，不硬编码

### Step 4：更新状态文件
- 🔴 必更：`进度追踪.md` / `99_当前状态卡.md` / `伏笔追踪.md`
- 🟡 按需：`角色状态.md` / `角色关系.md` / `资源账本.md` / `战力跟踪.md` / `时间线.md` / `章节分析.md` / `职业跟踪.md`
- 🟢 自动：`quality_trend.csv`

---

## 二、查 🔍（写完必查，精修靠模型）

### 检查脚本（只查不改）

```bash
# 字词级检查
python scripts/chapter_health_check.py --project <项目路径> --chapter <章节号>

# 结构级检查
python scripts/publish_readiness_audit.py --project <项目路径> --chapter <章节号>

# 综合决策（整章重写 OR 细修）
python scripts/rewrite_decider.py --project <项目路径> --chapter <章节号>
```

### 检查出问题 → 模型直接改

**不是跑修复脚本，而是模型读取问题后直接精修正文。**

精修优先级：
1. **最有效**：删跨章重复短语 / 替换AI模板词为口语 / 打断均匀句长
2. **次要**：加方言口语 / 加感官细节 / 删总结性心理段落
3. **容易忽略**：删AI元数据 / 统一文件名与标题

---

## 三、AI率检测（半自动）

> 全卷完成后批量检测，或关键章节抽查。

```bash
# 1. 提取文本
python scripts/ai_detect_online.py --project <项目路径> --chapter 1

# 2. 复制输出文本 → 粘贴到朱雀网页
#    https://matrix.tencent.com/ai-detect/ai_gen

# 3. 记录结果
python scripts/ai_detect_online.py --project <项目路径> --chapter 1 --record 15.2 --platform 朱雀

# 4. 查看汇总
python scripts/ai_detect_online.py --project <项目路径> --results
```

**标准**：< 20%优秀 / 20-40%可接受 / > 40%需精修

AI率高的章节 → 回到"查"步骤，模型精修后重新检测。

---

## 四、发 📤（全卷完成后）

### 发布前确认清单
- [ ] AI率检测 < 40%
- [ ] chapter_health_check 全部通过
- [ ] publish_readiness_audit 全部通过
- [ ] 读者三问已答

### 发布步骤
1. 导出txt到 `fanqie_auto_publish/chapters/`
2. 运行发布脚本（见 `fanqie_auto_publish/` 目录）
3. 发布后标记 `进度追踪.md` 发布状态

---

## 备份策略

- 每卷完成 → 存到 `备份文件/`
- 大修前后 → 存到 `备份文件/`
- 发布前 → 存到 `备份文件/`

---

## 不做的事

| 不做 | 原因 |
|------|------|
| 市场调研 | 固定题材，不跟风 |
| 跑修复脚本精修 | 模型直接改效果更好 |
| 本地AI率检测 | 不靠谱，必须用在线平台 |
| 简单同义词替换降重 | 统计特征不变，无效 |
