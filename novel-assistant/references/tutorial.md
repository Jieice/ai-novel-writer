# 进阶教程

## 一、反流水账规则

流水账就是平铺直叙、缺乏冲突、没有情绪起伏的叙述。

**避免方法1：增加冲突**
- 给流水账增添前因后果
- 增添故事冲突
- 增强故事内容的期待感

**避免方法2：加入强情绪**
- 让读者跟着你一起着急
- 让读者能跟你产生深度共鸣

**核心原则：**
- 一旦场面开始像日常记录，立刻补冲突或强情绪
- 纯过渡动作能删就删
- 不要一整章只交代"事情做成了"，要写过程里的压力和代价
- 章末要像电视剧一样留钩子

## 二、视角规则

### 写作人称选择

| 人称 | 特点 | 适用场景 |
| --- | --- | --- |
| 第一人称 | 代入感强、视角局限 | 新媒体风格、短篇 |
| 第三人称 | 视角自由、适合长篇 | 网文主流 |

### 视角使用原则
- 默认优先限制性第三人称
- 同一场景里不能乱切视角
- 多人场景把主要说话的人控制在三个以内

## 三、开篇规则

**问题1：开篇太寡淡**
- 从"炸楼"开始写，不要从"砌砖"开始

**问题2：开篇太复杂**
- 头三章1-3个场景就足够
- 开篇出现的人物3个就足够多

**问题3：开篇无钩子**
- 每一章结尾都留好悬念

## 四、钩子设计

**钩子类型：**
- 悬念钩子：读者想知道接下来会发生什么
- 伏笔钩子：埋下线索，读者期待揭晓
- 冲突钩子：矛盾即将爆发，读者期待结果
- 危机钩子：主角陷入困境，读者期待解决

**设置原则：**
- 每一章结尾都要有钩子
- 钩子可大可小
- 悬念要留得恰到好处

## 五、欲望与好奇心

### 基础欲望（被动欲望）
- 不劳而获、高人一等、权势地位

### 主动欲望（期待感）
- 反派登门作恶，读者期待打脸反转
- 主角隐藏实力，读者期待爆发时刻
- 误会即将揭开，读者期待真相大白

### 情绪缺口
- 憋屈的对立情绪就是扬眉吐气
- 情绪释放爆发的时刻要超过读者的心理预期

## 六、五感代入

**五感描写：**
- 视觉：看到的画面
- 听觉：听到的声音
- 嗅觉：闻到的气味
- 触觉：摸到的质感
- 味觉：尝到的味道

**使用原则：**
- 在大的场景塑造下适当加入一到两种五感描述
- 通过文字描述增强读者的画面感

## 七、增强代入感的四种方式

1. **熟悉感**：熟悉的物品、口头禅、场景
2. **标签**：人设的标签要提前构思好（小缺点、特质、外貌特征）
3. **冲突的紧迫性**：危机感拉满，读者被剧情引导着推着往前走
4. **接地气**：依靠人物缺点，白话不高深不费脑

## 八、人设规则

### 人设扁平化问题
- 所有角色说话语气不能一样
- 所有角色发脾气不能同一种模式
- 角色前后人设必须统一

### 改进方法
- 通过事件发展描写人物的行为方式和情绪表达
- 人设的特点在故事情节里突显出来

## 九、11种创作模式

| 模式 | 用途 |
| --- | --- |
| 设定模式 | 补设定、梳理派系、搭世界规则 |
| 大纲模式 | 主线拆解、阶段目标、冲突安排 |
| 章节模式 | 直接写正文、场景、章节 |
| 改写模式 | 保留原意、去冗余、去AI味 |
| 文本优化模式 | 分析原稿、保留作者风格、提供多版本 |
| AI去味模式 | 识别AI痕迹、改得更自然 |
| 创意导师模式 | 指定口吻下重写、增强情绪张力 |
| 朴素讲故事模式 | 普通讲故事人心态、轻松直接有趣 |
| 审稿模式 | 先指出问题、再给修改建议 |
| 包装模式 | 书名、简介、卖点、开篇角度 |
| 考据模式 | 核查时代、地点、人物、政策 |
