# 小说创作助手 skill

> 三件事：写、查、发。精修靠模型能力，不靠脚本正则。

---

## 三大核心功能

### 一、写 ✍️

写正文两阶段流程：

**阶段一：写初稿**

1. **读上下文**：`99_当前状态卡.md` + 前章最后500字
2. **构思**（动笔前必填）：冲突点/主角挫折/反派行动/金手指代价/章末钩子/场戏规划
3. **写初稿**（第一人称 + 朴素讲故事模式）：
   - **必须第一人称**（"我"不是"陈大山"）——这是降重的关键
   - 心态：你不是在炫技，不是在完成3000字任务。你只是一个喜欢观察生活的人在讲故事
   - 节奏：不急着推进，允许在一个动作/画面/感觉上多停一会儿，像雕东西不像赶工
   - 语言：直接，少修饰，多动作，多写"正在发生什么"，口语化优先
   - 句式：长短句自然混用，不要每一句都像同一个模具倒出来的
   - 段落：短，透气，适合手机阅读，每段只承载一个核心点
   - 禁忌：禁止文学腔、禁止装深沉、禁止为了完成字数而写
   - 对话：独立成段+双引号，展示不讲述

**阶段二：风格润色**

4. **读风格指南**：`山村小神医/项目文件/风格指南.md`（8条铁律+人味写法）
5. **按8条铁律润色**：
   - ①允许丢人：每章至少1处主角丢人时刻
   - ②废话=人味：每5-8段加1处没功能的信息
   - ③朋友废话：对话至少1/3是废话/起哄
   - ④笨情绪：不说"愤怒""悲伤"，直接说"不知道""害怕"
   - ⑤朴素过渡：只用因为/所以/然后/过了
   - ⑥人物出场：位置+动作就完了
   - ⑦节奏不均：慢的地方可以很慢，快的一句话带过
   - ⑧内心独白：不超过15字，用口语
6. **更新状态文件**（按优先级）：
   - 🔴 必更：`进度追踪.md` / `99_当前状态卡.md` / `伏笔追踪.md`
   - 🟡 按需：`角色状态.md` / `角色关系.md` / `资源账本.md` / `战力跟踪.md` / `时间线.md` / `章节分析.md` / `职业跟踪.md`
   - 🟢 自动：`quality_trend.csv`

### 二、查 🔍

写完后必跑检查，**精修由模型直接改，不跑修复脚本**。

#### 检查脚本（只查不改）

| 脚本 | 用途 | 什么时候跑 |
|------|------|------------|
| `chapter_health_check.py` | 字词级：引号/模板词/口语化/冲突密度 | 每章写完 |
| `publish_readiness_audit.py` | 结构级：场景数/裸对话/账本句/总结句 | 每章写完 |
| `rewrite_decider.py` | 综合决策：整章重写 OR 细修 | 检查前两项后 |

```bash
python scripts/chapter_health_check.py --project <项目路径> --chapter <章节号>
python scripts/publish_readiness_audit.py --project <项目路径> --chapter <章节号>
python scripts/rewrite_decider.py --project <项目路径> --chapter <章节号>
```

#### AI率检测

> ⚠️ 本地脚本的AI检测不可靠（TTR+句长CV ≠ 真正的困惑度/突发性模型），必须用在线平台。

**检测流程（半自动）**：

```bash
# 第1步：提取章节文本
python scripts/ai_detect_online.py --project <项目路径> --chapter <章节号>

# 第2步：复制输出文本，粘贴到朱雀网页检测
#        https://matrix.tencent.com/ai-detect/ai_gen

# 第3步：记录检测结果
python scripts/ai_detect_online.py --project <项目路径> --chapter <章节号> --record <AI率> --platform 朱雀

# 查看所有章节检测汇总
python scripts/ai_detect_online.py --project <项目路径> --results
```

**检测平台**：
1. **天眼AI**（首选）— https://www.tianyanai.org/
   - 完全免费，实测有效，第一人称+8条铁律写法在此平台验证AI率降低
2. **朱雀AIGC检测**（备选）— https://matrix.tencent.com/ai-detect/ai_gen
   - 腾讯混元安全团队，中文准确率95%+，免费，但对第一人称+人味写法的识别更严格

**检测标准**：
- AI率 < 20%：✅ 优秀
- AI率 20-40%：⚠️ 可接受
- AI率 > 40%：❌ 需精修

#### 精修降重策略（第一人称+8条铁律，天眼AI实测有效）

> ⚠️ 朱雀实测CH1=100%的教训：第三人称AI无论怎么精修/重写/换词/打断句长，朱雀都能识别（AIGC=0.9984）。
> ✅ 天眼AI实测验证：**第一人称+风格指南8条铁律，AI率明显降低**。
> 关键：不是"改词"是"换写法"——从写作方式本身就模拟人。

**润色核心流程**：

1. **读风格指南**：`山村小神医/项目文件/风格指南.md`
2. **确认第一人称**：全文"我"不是"陈大山"
3. **逐段对照8条铁律润色**：丢人/废话/笨情绪/朴素过渡/简略出场/节奏不均/短独白
4. **核对信息**：保证原意和关键情节不走样
5. **全文微调**：通读一遍，看节奏、断句、段落和整体气质是否顺

**已验证的手法有效性**：

| 有效性 | 手法 | 说明 |
|--------|------|------|
| ✅ 有效 | 第一人称+8条铁律 | 天眼AI实测AI率降低 |
| ✅ 有效 | 允许丢人/废话/笨情绪 | 改变叙事逻辑本身 |
| ✅ 有效 | 节奏不均/朴素过渡 | 打破AI的规整性 |
| ❌ 无效 | 第三人称+打断句长 | 朱雀仍判100% |
| ❌ 无效 | 第三人称+加感官/删模板词 | 朱雀仍判100% |
| ❌ 无效 | 同义词替换/改标点 | 统计特征不变 |

## 六条铁律（硬阈值）

| 编号 | 规则 | 阈值 |
|------|------|------|
| **铁律1** | 对话必须 `""` 独立成段 | 引号对 ≥ 6 |
| **铁律2** | 每章主角必遇挫折 | 挫折类型六选一 |
| **铁律3** | 用灵珠必写代价 | 代价关键词 ≥ 1 |
| **铁律4** | 一章 ≤ 2 场戏 | 小标题 ≤ 2 |
| **铁律5** | 反派必有实质行动 | 不能只口头威胁 |
| **铁律6** | 禁用AI模板词 | 套路词 ≤ 8/章 |

**挫折类型**：行动挫折/人际挫折/损失挫折/意外挫折/能力挫折/金手指副作用

**代价关键词**：疲惫/累得/虚脱/脸色发白/腿软/发抖/发烧/眼前发黑/冷汗/打哆嗦

**禁用词**：笑了笑/点了点头/心里头一紧/等着吧/十倍奉还 等

## 读者三问（交付前必答）

1. **这章我看完记住了什么？**（记不住 = 场景太多太浅）
2. **主角台词换个角色说合适吗？**（合适 = 角色没立起来）
3. **这段我在第几章读过？**（读过 = 中二独白/心理套路在复读）

## 项目文件模板（中文命名）

> 开新书时自动获得，详见 `00_项目模板/项目文件/文件索引模板.md`

### 标准文件（14个）

| 文件 | 用途 |
|------|------|
| `小说大纲.md` | 核心设定、故事大纲 |
| `人物设定.md` | 主角/配角/反派设定（含职业+组织） |
| `弧线摘要.md` | 每弧摘要，写前必读 |
| `章节分析.md` | 每章分析记录 |
| `角色状态.md` | 角色心理/存活状态+章节号防覆盖 |
| `角色关系.md` | 人物关系图+亲密度追踪 |
| `资源账本.md` | 现金/药材/物资追踪 |
| `战力跟踪.md` | 战力等级/金手指状态 |
| `伏笔追踪.md` | 伏笔状态/强度/超期检测 |
| `剧情线索.md` | MT线索管理 |
| `时间线.md` | 故事时间/天气/地点 |
| `进度追踪.md` | 章节完成状态（当前章节号从此文件获取） |
| `职业跟踪.md` | 角色职业阶段+技能等级 |
