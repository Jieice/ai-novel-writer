# 读者留存指标库

> 基于500本扑街书拆解与平台数据验证，收录可复用的读者留存模板

## 一、开篇200字抓人模板

### 核心原则
- 前200字无强钩子，80%用户直接划走
- 主角平均200字内出场
- 困境+翻盘承诺同步落地

### 模板1：冲突直接爆发型
```
[第一句：冲突/危机直接呈现]
[第二句：主角立即反应]
[第三句：金手指/能力暗示]

示例：
"你爷爷的手札，交出来。"
刀架在脖子上的时候，陈大山才明白，爷爷留给他的那本破书，不是什么废纸。
他摸了摸胸口，那里有一枚冰凉的珠子，正在微微发热。
```

### 模板2：反常事件型
```
[第一句：颠覆认知的反常事件]
[第二句：主角困惑/震惊]
[第三句：悬念钩子]

示例：
陈大山推开爷爷的房门，看见一个本该死去三年的人，正坐在书桌前翻看那本手札。
"你爷爷没告诉你？"那人抬起头，"这手札，本来就是我的。"
```

### 模板3：困境+翻盘承诺型
```
[第一句：主角困境]
[第二句：外界压力/嘲讽]
[第三句：金手指激活/翻盘暗示]

示例：
"陈大山，你个废物，连你爹的咳嗽都治不好，还敢说自己是中医大专毕业的？"
赵老三的嘲讽声在院子里回荡，围观村民有的摇头，有的窃笑。
陈大山没说话，只是摸了摸口袋里那枚刚激活的灵珠。
```

### 模板4：信息差型
```
[第一句：读者知道主角不知道的信息]
[第二句：主角行动（读者有期待感）]
[第三句：即将揭晓的悬念]

示例：
镇长不知道，他即将治好的那位"神秘病人"，正是害他父亲的仇人的父亲。
"周老说这病人很特殊，让我务必小心。"陈大山跟着镇长上了车。
车窗外，赵老三正站在路边，嘴角带着一丝诡异的笑。
```

## 二、章中钩子模板

### 核心原则
- 每800-1200字一个钩子
- 章中3-4个小钩子可让留存率提升30%
- 确保读者全程保持注意力

### 类型1：微型反转钩
```
[铺垫：情况看似正常]
[反转：突然变化]
[悬念：为什么？]

示例：
陈大山正要开口，却发现老中医的手在微微发抖。
不对劲。
一个治了一辈子病的老中医，怎么会手抖？
```

### 类型2：信息差钩
```
[读者视角：知道某信息]
[角色视角：不知道]
[期待：何时发现？]

示例：
陈大山不知道的是，就在他治病的这个晚上，赵老三已经带人去了他家。
王大娘正一个人在家，听着院外的脚步声，手心全是汗。
```

### 类型3：新疑问钩
```
[发现新线索/信息]
[产生疑问]
[悬念：答案是什么？]

示例：
"你爷爷的灵珠，我知道在哪。"
陈大山愣住了："你知道？"
老人没回答，只是从抽屉里拿出一张泛黄的照片，上面是一个年轻人，手里拿着一枚和陈大山一模一样的珠子。
"这是谁？"
"你爷爷。"
```

### 类型4：情绪波动钩
```
[角色反应异常]
[读者好奇：为什么？]
[悬念：背后原因]

示例：
林秀梅突然低下了头，耳根子有点热。
陈大山注意到她的反应，心里头有些异样的感觉。
"秀梅，你怎么了？"
"没……没什么。"她转过身去，声音很小。
```

### 类型5：预兆钩
```
[异常现象]
[角色困惑]
[悬念：意味着什么？]

示例：
灵珠突然开始发热，比以往任何一次都强烈。
陈大山停下脚步，灵珠指向的方向，是一家看起来很普通的药材铺。
那里有什么？
```

## 三、章尾悬念模板

### 核心原则
- 带明确悬念的章节，追读率平均比平铺直叙高22%
- 必须让读者必须点开下一章才能知道答案
- 所有钩子必须在后续剧情中兑现

### 类型1：纵向钩（悬念钩）
```
[揭晓前一秒戛然而止]
[读者必须点下一章]

示例：
"其实，你才是那个……"
话音未落，门被踹开了。
```

```
"你爷爷当年把灵珠托付给我，让我保护好它。"
老人顿了顿，眼神变得复杂："但他说了一句话，我到现在都没明白。"
"什么话？"
"他说：'如果有一天，有人来拿这枚灵珠，先问他一个问题。'"
"什么问题？"
老人看着陈大山："你愿意为它付出什么代价？"
```

### 类型2：横向钩（爽点钩）
```
[爽点完成]
[立刻抛出更大的新危机]

示例：
赵老三跪在地上，当众认错。
陈大山松了口气，这场风波终于结束了。
就在这时，他注意到人群中，有一个陌生的面孔，正死死盯着他。
那人嘴角微微扬起，转身离开了。
```

```
野山参到手了，灵珠升级成功。
陈大山感觉前所未有的强大。
周济民拍着他的肩膀说："以后省城的事，我罩着你。"
就在这时，灵珠突然指向一个方向——在省城某处，有另一个灵珠。
```

### 类型3：人物钩（关系钩）
```
[用情绪张力代替外部事件]
[读者想知道接下来会发生什么]

示例：
"不用了，我明白了。"
林秀梅转身就走，眼眶有点红。
陈大山想追上去，却发现自己的腿像灌了铅一样，迈不动。
他到底做错了什么？
```

```
"我愿意留在省城。"
林秀梅说完这句话，没敢看陈大山。
陈大山愣了一下，然后笑了："那正好，我也打算在省城发展。"
林秀梅抬起头，眼眶有点红，但嘴角是笑的。
```

## 四、情绪闭环设计模板

### 核心原则
- 每章必须完成一次完整的情绪闭环
- 结构：压抑→反转→爽感→新期待
- 即使是铺垫章节，也要有小的情绪波动

### 模板1：标准情绪闭环
```
[压抑：主角遭遇困境]
[反转：出现转机]
[爽感：困境解决]
[新期待：新的悬念]

示例（CH33）：
压抑：野山参被偷，老中医急病复发
反转：灵珠突然发热，指向野山参位置
爽感：找到野山参下落
新期待：张二狗出现在仓库门口
```

### 模板2：双情绪闭环
```
[小闭环：压抑→反转→小爽]
[大闭环：新压抑→大反转→大爽→新期待]

示例（CH36）：
小闭环：仓库突袭→顺利进入→找到野山参
大闭环：李万山残党反抗→周济民出手→残党被抓→灵珠升级→感知到另一个灵珠
```

### 模板3：铺垫章节的情绪闭环
```
[微压抑：小困难]
[微反转：小发现]
[微爽：小进展]
[新期待：新悬念]

示例（CH37）：
微压抑：灵珠新能力需要测试
微反转：发现可以感知药材品质
微爽：周济民提出合作
新期待：感知到奇怪的能量波动
```

### 模板4：高潮章节的情绪闭环
```
[大压抑：最大困境]
[大反转：关键转折]
[大爽：彻底胜利]
[大期待：下一阶段悬念]

示例（CH27-28）：
大压抑：赵老三嚣张，陈大山被打
大反转：镇长站队，公开保护
大爽：赵老三认罪，赔钱道歉
大期待：李万山还没解决
```

## 五、打脸铺垫比例模板

### 核心原则
- 3:1是新手安全线（3份铺垫+1份打脸）
- 铺垫≠憋屈，要"伪压抑，真期待"
- 在铺垫中穿插微爽点

### 模板1：标准3:1打脸
```
[第1章：反派挑衅，主角隐忍]
[第2章：反派得寸进尺，外界不看好]
[第3章：主角蓄力，拿到破局关键]
[第4章：瞬间爆发，极致打脸]

示例（CH24-27）：
CH24：赵老三嚣张，抓人质
CH25：赵老三举报，陈大山被禁
CH26：镇长父亲发病，陈大山再救
CH27：公开揭露赵老三罪行
```

### 模板2：快节奏2:1打脸
```
[第1章：反派挑衅]
[第2章：主角直接反击]

适用场景：
- 免费平台短篇
- 新媒体文
- 女频重生复仇

示例（CH5-6）：
CH5：赵老三伪造地契，威胁陈大山
CH6：陈大山查档案，证明伪造，赵老三吃瘪
```

### 模板3：长线布局5:1打脸
```
[第1-5章：多章节铺垫，埋伏笔]
[第6章：一次性集中爆发]

适用场景：
- 付费平台长篇
- 男频玄幻修仙
- 大高潮设计

示例（CH13-18）：
CH13-17：赵老三步步紧逼，恒昌断销路，林秀梅被威胁，北坳被烧
CH18：陈大山找到周济生，获得县城人脉，开始反击
```

### 模板4：伪压抑真期待
```
[铺垫：反派嚣张]
[读者视角：知道主角有底牌]
[期待：看反派怎么死]

示例：
赵老三不知道，陈大山口袋里的灵珠，已经感知到了他藏在仓库里的赃物。
"陈大山，你给我等着！"赵老三恶狠狠地说。
陈大山笑了笑，没说话。
他已经在想，等镇长知道了这些，赵老三会是什么表情。
```

## 六、爽点密度设计模板

### 核心原则
- 每3章内出现一次明确爽点
- 爽点阶梯式升级，每次比上一次更高
- 爽点类型多样化

### 爽点类型清单

| 类型 | 定义 | 示例 |
|------|------|------|
| 技术爽 | 医术/能力展示 | 治好省城大夫治不了的病 |
| 人脉爽 | 大人物站队 | 镇长公开保护 |
| 打脸爽 | 反派被打脸 | 赵老三当众认错 |
| 升级爽 | 金手指升级 | 灵珠升级至二级 |
| 收获爽 | 资源增加 | 获得百年野山参 |
| 感情爽 | 感情升温 | 林秀梅愿意同行 |
| 见识爽 | 开阔眼界 | 省城见世面 |
| 事业爽 | 事业进展 | 镇上开铺子 |
| 真相爽 | 揭开谜团 | 爷爷往事揭晓 |
| 意外爽 | 意外收获 | 第九种药材意外到手 |

### 爽点阶梯升级示例
```
第一卷（CH1-20）：
- 小爽：治好父亲咳嗽
- 中爽：赶集赚第一桶金
- 大爽：获得周济生引荐

第二卷（CH21-30）：
- 小爽：治好镇长父亲
- 中爽：镇长公开站队
- 大爽：赵老三认罪赔偿

第三卷（CH31-40）：
- 小爽：省城见世面
- 中爽：灵珠升级
- 大爽：拿回爷爷的灵珠，揭开真相
```

## 七、期待感管理模板

### 核心原则
- 保持至少两种期待：大期待+小期待
- 大期待贯穿全卷，小期待每章推进
- 期待兑现后立刻铺设新期待

### 大期待设计
```
定义：贯穿多章节的核心悬念

示例：
第一卷：能否在县城站稳脚跟？
第二卷：能否打败赵老三？
第三卷：能否升级灵珠？爷爷的灵珠在哪？

特征：
- 读者持续追更的核心动力
- 每章都在推进
- 兑现时是高潮
```

### 小期待设计
```
定义：单章内可推进或兑现的悬念

示例：
CH31：匿名信是谁写的？
CH32：老中医认识爷爷？
CH33：灵珠为什么发热？

特征：
- 每章都有
- 快速兑现或推进
- 兑现后立刻铺设新期待
```

### 期待感断裂预警
```
危险信号：
- 连续3章没有推进大期待
- 小期待兑现后没有新期待
- 大期待兑现后没有新的大期待

解决方案：
- 检查每章是否推进主线
- 章末必须有钩子
- 大高潮后立刻铺设新目标
```

## 九、弃书雷区预警

### 高危雷区
| 雷区 | 表现 | 解决方案 |
|------|------|----------|
| 开篇慢热 | 前2000字铺垫背景 | 直接切入冲突 |
| 爽点真空 | 连续3章无爽点 | 加入小爽点 |
| 主角掉线 | 主角超20%篇幅不出场 | 增加主角戏份 |
| 设定轰炸 | 第一章超15个陌生名词 | 分散到多章 |
| 主线跑偏 | 大量无关支线 | 删除或压缩 |
| 期待断裂 | 大期待无推进 | 每章推进主线 |
| 人设崩塌 | 主角行为矛盾 | 检查行为逻辑 |
| 断更 | 更新不稳定 | 保持日更 |
