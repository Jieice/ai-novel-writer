# 小说项目模板

本文件夹包含小说创作项目的标准文件结构，开新小说时复制此文件夹并修改内容即可。

## 文件结构

```
小说名称/
├── 项目文件/
│   ├── 小说大纲.md              # 核心设定、故事大纲、章节规划
│   ├── 人物设定.md              # 主角、配角、反派详细设定
│   ├── 99_当前状态卡.md         # 当前进度、主角状态、创作追踪
│   ├── 进度追踪.md              # 章节完成进度、脚本检查状态
│   ├── 文件索引.md              # 所有文件索引、快速查找
│   ├── 读者留存指标库.md        # 开篇200字、章中钩子、章尾悬念模板
│   ├── 伏笔追踪.md             # 伏笔状态、强度、期待感评分
│   ├── 弧线摘要.md             # 弧线摘要
│   ├── 章节分析.md             # 章节分析记录
│   ├── 角色关系.md             # 动态人物关系图
│   ├── 角色状态.md             # 角色心理/存活状态+章节号防覆盖
│   ├── 资源账本.md             # 现金/药材/物资追踪
│   ├── 战力跟踪.md             # 战力等级/金手指状态
│   ├── 剧情线索.md             # MT线索管理
│   ├── 时间线.md               # 故事时间/天气/地点
│   ├── 职业跟踪.md             # 角色职业阶段+技能等级
│   ├── 组织关系.md             # 组织势力+成员+据点
│   ├── 第一卷节奏规划.md       # 第1-10章节奏规划
│   ├── 第二卷节奏规划.md       # 第11-20章节奏规划
│   └── 第三卷节奏规划.md       # 第21-30章节奏规划
├── 正文卷/
│   └── 第X章_*.md             # 章节正文文件
└── README.md                  # 本文件
```

## 使用流程

### 1. 开新项目

使用 novel-assistant skill：
1. 说"开新书"触发创建流程
2. 确认小说类型、标题、核心卖点
3. 自动复制模板并创建文件
4. 可选：说"学习风格"分析参考书

### 2. 写正文

使用 novel-assistant skill：
1. 说"继续写"或"写第X章"
2. AI自动读取状态卡、任务计划
3. 按8条铁律写作（第一人称+对话引号+口语化+展示不讲述）
4. 自动执行内容检查（毒点/战力/时间线/一致性）
5. 自动更新所有项目文件

### 3. 检查与优化

使用 novel-assistant skill：
1. 说"检查"执行降重+更新+AI检测+毒点检测
2. 说"冲突规划"规划未来10-20章冲突分布
3. 说"市场"分析热门题材和收益

### 4. 发布

使用 novel-assistant skill：
1. 说"发布"导出章节为txt
2. 自动放入番茄发布目录

## 8条铁律（写作规范）

| 铁律 | 要求 |
| --- | --- |
| 1. 对话双引号 | 对话独立成段 + 双引号 |
| 2. 禁止AI模板词 | 禁止"笑了笑/点了点头/深吸一口气"等AI套路 |
| 3. 口语化词 | 使用"俺/咱/啥/咋/呗/嘛"等乡土方言 |
| 4. 主角挫折 | 每章必须有主角挫折场景 |
| 5. 反派实质行动 | 反派必须有实质威胁行动 |
| 6. 金手指代价 | 金手指使用必须有代价（体力消耗/虚弱） |
| 7. 时代细节 | 融入90年代乡土细节 |
| 8. 展示不讲述 | 用动作/细节/对话展示情绪，禁止直接讲述 |

## 注意事项

1. **资源同步**：每章写完后必须更新 `资源账本.md`，避免资源不连续
2. **伏笔追踪**：埋下伏笔时记录到 `伏笔追踪.md`，回收时标记状态
3. **人物关系**：关系变化时更新 `角色关系.md`
4. **战力跟踪**：金手指/能力变化时更新 `战力跟踪.md`
5. **时间线**：确保故事时间连续，检查天气合理性
6. **8条铁律**：写作时直接应用8条铁律（第一人称+对话引号+口语化）
7. **考据准确性**：年代文需要严格考据
