# 小说项目模板

本文件夹包含小说创作项目的标准文件结构，开新小说时复制此文件夹并修改内容即可。

## 文件结构

```
小说名称/
├── 项目文件/
│   ├── 小说大纲.md                # 核心设定、故事大纲、章节规划
│   ├── 人物设定.md                # 主角、配角、反派详细设定
│   ├── 99_当前状态卡.md           # 当前进度、主角状态、创作追踪
│   ├── task_plan.md               # 当前阶段任务计划
│   ├── progress.md                # 章节完成进度追踪
│   ├── pending_hooks.md           # 伏笔和线索追踪
│   ├── resource_ledger.md         # 资源账本（现金、物资、人脉、能力）
│   ├── character_relationships.md # 动态人物关系图
│   ├── power_tracking.md          # 战力/金手指跟踪
│   ├── timeline.md                # 时间线（故事时间、天气、地点）
│   ├── quality_report.md          # 质量报告（章节评估、风险提示）
│   ├── findings.md                # 考据记录（年代/历史事实）
│   └── file_index.md              # 文件索引
├── 正文卷/
│   └── 第1章_*.md                 # 章节正文文件
└── README.md                      # 本文件
```

## 文件说明

### 核心文件（必须）

| 文件 | 用途 | 更新频率 |
| --- | --- | --- |
| 小说大纲.md | 设定总索引：核心设定、故事简介、人物小传、章节规划 | 项目开始时 |
| 人物设定.md | 主角、配角、反派的详细设定（背景、性格、金手指、成长弧光） | 项目开始时 |
| 99_当前状态卡.md | 当前时间点、主角状态、金手指状态、人物关系、资源边界、创作追踪 | 每章更新 |
| task_plan.md | 当前阶段的任务计划，每章的目标、内容、冲突、爽点 | 每章更新 |
| progress.md | 章节完成状态、字数、里程碑、资源追踪、感情线进度 | 每章更新 |
| pending_hooks.md | 已埋下的伏笔、待展开的线索、悬念钩子 | 每章更新 |

### 追踪文件（每章更新）

| 文件 | 用途 |
| --- | --- |
| resource_ledger.md | 资源账本：现金收支、药材库存、物资、人脉、能力 |
| character_relationships.md | 动态人物关系图：关系变化追踪 |
| power_tracking.md | 战力跟踪：金手指等级、技能变化、能力合理性 |
| timeline.md | 时间线：故事日期、事件、天气、地点、时间连续性 |
| quality_report.md | 质量报告：章节评分、节奏评估、毒点检测、风险提示 |

### 辅助文件（按需更新）

| 文件 | 用途 |
| --- | --- |
| findings.md | 考据记录：年代物价、收入、生活细节、方言 |
| file_index.md | 文件索引：快速查找所有文件 |

## 使用流程

### 1. 开新项目

使用 novel-assistant skill：
1. 说"开新书"触发创建流程
2. 确认小说类型、标题、核心卖点
3. 自动复制模板并创建文件
4. 可选：说"学习风格"分析参考书

### 2. 写正文

使用 novel-assistant skill：
1. 说"继续写"或"写第X章"
2. AI自动读取状态卡、任务计划
3. 按BypassAIGC四阶段降重策略写作
4. 自动执行内容检查（毒点/战力/时间线/一致性）
5. 自动更新所有项目文件

### 3. 检查与优化

使用 novel-assistant skill：
1. 说"检查"执行降重+更新+AI检测+毒点检测
2. 说"冲突规划"规划未来10-20章冲突分布
3. 说"市场"分析热门题材和收益

### 4. 发布

使用 novel-assistant skill：
1. 说"发布"导出章节为txt
2. 自动放入番茄发布目录

## 注意事项

1. **资源同步**：每章写完后必须更新 `resource_ledger.md`，避免资源不连续
2. **伏笔追踪**：埋下伏笔时记录到 `pending_hooks.md`，回收时标记状态
3. **人物关系**：关系变化时更新 `character_relationships.md`
4. **战力跟踪**：金手指/能力变化时更新 `power_tracking.md`
5. **时间线**：确保故事时间连续，检查天气合理性
6. **质量报告**：定期评估章节质量，关注风险提示
7. **降重策略**：写作时直接应用BypassAIGC四阶段（润色→增强→特殊场景→整体）
8. **考据准确性**：年代文需要严格考据，记录到 `findings.md`
