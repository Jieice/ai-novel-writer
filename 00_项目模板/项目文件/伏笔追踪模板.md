# 伏笔追踪

> 借鉴 MuMuAINovel 伏笔系统，支持评分、超期检测、章节号防覆盖

---

## 状态定义

| 状态 | 含义 | 状态转换 |
|------|------|---------|
| ⏳ pending | 待埋入（已规划但未写入章节） | → planted |
| 🌱 planted | 已埋入（已在章节中埋下） | → resolved / abandoned |
| ✅ resolved | 已回收 | - |
| 🔸 partially_resolved | 部分回收（长线伏笔分多次） | - |
| ❌ abandoned | 已废弃 | - |

---

## 评分系统

| 维度 | 范围 | 说明 |
|------|------|------|
| **强度** | 1-10 | 影响读者多强烈感知到此伏笔 |
| **隐藏度** | 1-10 | 越高越隐蔽，读者越难察觉 |
| **紧急度** | 0-3 | 0=不紧急, 1=需关注, 2=急需回收, 3=已超期 |

---

## 章节号防覆盖机制

**核心原则**：更新伏笔时，新章节号必须 ≥ 已记录的章节号，否则跳过更新

```
防覆盖规则：
- 心理状态：state_updated_chapter 新值 < 旧值 → 跳过
- 伏笔埋入：plant_chapter_number 新值 < 旧值 → 跳过
- 伏笔回收：actual_resolve_chapter_number 新值 < 旧值 → 跳过
```

---

## 超期检测规则

- **OVERDUE**：当前章 > 计划解决章 + 5
- **URGENT**：must-resolve 且 距计划章 ≤ 3

---

## 优先级定义

- **must-resolve**：必须在指定章节前解决（影响主线）
- **should-resolve**：应该解决（影响读者体验）
- **optional**：可解决可不解决（锦上添花）
